## vanila css

### 해당 component가 있는 폴더에 css 파일을 만들어서 스타일링 하기

#### 장점

- 다른 사람이 스타일링 작업을 하여 전달해줄 수 있다.

#### 단점

- 각각의 컴포넌트와 css 파일이 스코핑 되어 있지 않는다. 그렇기 때문에 충돌이 발생할 수 있다. (css 파일을 컴포넌트에 맞게 여러 개로 나누어도 충돌한다는 의미)

#### 해결방법

- 인라인 스타일 적용

</br>

## inLine style

### jsx 태그 안에 `style ={{}}` 으로 스타일을 적용할 수 있다.

#### 장점

- 내가 적용하고자 하는 jsx 코드에만 영향을 미친다.
- 스타일 적용이 쉽고 간단하다.

#### 단점

- 여러 개의 컴포넌트와 jsx 코드에 적용을 하거나 수정을 하려면 일일이 다 바꿔줘야 한다.
- css와 jsx 코드에 구분이 없다는 것이다.
- 다른 사람과 작업을 하기 불편하다.

```jsx
<p>
  <label>Email</label>
  <input
    type="email"
    style={{ backgroundColor: emailNotValid ? "#fed2d2" : "#d1d5db" }}
    // className={emailNotValid ? 'invalid' : undefined}
    onChange={(event) => handleInputChange("email", event.target.value)}
  />
</p>
```

이렇게 인라인 스타일을 이용해 조건적으로 스타일을 적용시킬 수 있다.

</br>

## 클래스

```jsx
<input
            type="email"
             {/* style={{ backgroundColor: emailNotValid ? "#fed2d2" : "#d1d5db" }} */}
            className={emailNotValid && "invalid"}
            onChange={(event) => handleInputChange("email", event.target.value)}
          />
```

이런식으로 작성하게 되면 false일때 emailNotValid로 선언되어서 에러가 발생하게 된다.

```jsx
<input
  type="email"
   {/* style={{ backgroundColor: emailNotValid ? "#fed2d2" : "#d1d5db" }} */}
  className={emailNotValid ? "invalid" : undefined}
  onChange={(event) => handleInputChange("email", event.target.value)}
/>
```

그러므로 이런식으로 작성을 해야 에러가 발생하지 않게 된다.

```jsx
<label className="label invalid">Email</label>
```

이런식으로 작성이 가능하고

```jsx
<label className={`label ${emailNotValid ? "invalid" : ""}`}>Email</label>
```

이렇게도 작성이 가능하다

## CSS 모듈

React CSS Modules는 CSS의 범위를 컴포넌트 단위로 제한하여 스타일링 충돌을 방지하고, CSS를 모듈화하여 관리하는 방법이다. CSS Modules는 일반적인 CSS와 유사하게 작성되지만, 각각의 CSS 클래스와 ID가 고유하게 생성되어 컴포넌트 간의 스타일 충돌을 방지한다. 관련 컴포넌트 CSS 파일에 `.module.css`로 파일명을 작성하면 적용이 된다. 이러한 css 파일에서 가져온 클래스는 고유한 해시를 포함한 이름으로 변환된다.

### CSS 모듈의 특징

- 각 컴포넌트의 스타일이 해당 컴포넌트 파일과 함께 관리되므로, 스타일을 유지보수 하기가 쉽다. 컴포넌트와 스타일이 논리적으로 묶여 있어 파일 구조가 깔끔해진다.

- CSS 파일을 컴포넌트 폴더 안에 배치하여 해당 컴포넌트와 직접적으로 연관된 스타일임을 명확히 할 수 있다.

- JavaScript 내에서 동적으로 클래스명을 설정할 수 있다. 이를 통해 조건부 스타일링이나 여러 클래스를 쉽게 결합할 수 있다.

```jsx
import logo from "../assets/logo.png";
import classes from "./Header.module.css";

export default function Header() {
  return (
    <header>
      <img src={logo} alt="A canvas" />
      <h1>ReactArt</h1>
      <p className={classes.paragraph}>
        A community of artists and art-lovers.
      </p>
    </header>
  );
}
```

### CSS 모듈의 장점

- CSS 클래스명이 자동으로 고유하게 변환되므로, 전역 네임스페이스 오염 없이 컴포넌트 단위로 스타일을 관리할 수 있다.

- 스타일이 컴포넌트와 함께 묶여 있어, 코드가 더 모듈화되고 관리하기 쉬워진다. 이는 특히 대규모 애플리케이션에서 유용하다.

- JavaScript 내에서 조건부로 클래스를 쉽게 적용할 수 있어, 동적 스타일링이 편리하다.

- 전역 CSS를 사용하지 않아, 한 컴포넌트의 스타일이 다른 컴포넌트에 영향을 미치는 것을 방지할 수 있다.

### CSS 모듈의 단점

- CSS Modules는 스타일을 외부 CSS 파일에 작성하므로, CSS-in-JS와 비교했을 때 다소 유연성이 떨어질 수 있다. CSS-in-JS는 스타일을 JavaScript내에 직접 작성하여 더 많은 동적 스타일링 옵션을 제공한다.

- CSS Modules를 사용하려면 빌드 도구의 지원이 필요하다. Create React App(CRA)과 같은 도구를 사용하면 설정이 간단하지만, 빌드 도구 설정을 직접 관리하는 경우에는 추가 설정이 필요할 수 있다.

</br>

## styled-components 패키지

React 애플리케이션에서 CSS를 컴포넌트 수준에서 관리할 수 있게 해주는 유명한 CSS-in-JS 라이브러리이다. 스타일을 자바스크립트 파일 내에서 정의하고, 컴포넌트와 함께 결합하여 관리할 수 있다. 이를 통해 스타일의 범위가 컴포넌트 제한되고, 글로벌 스타일 충돌을 방지할 수 있다.

### styled-components의 특징

- 스타일을 자바스크립트 파일 내에서 작성하여, 컴포넌트와 스타일을 하나의 파일에서 관리할 수 있다.

- ES6의 태그된 템플릿 리터럴을 사용하여 스타일을 정의한다. 이는 스타일을 직관적이고 가독성 있게 작성할 수 있게 한다.

- 필요한 CSS만 렌더링하여 페이지 로딩 속도를 향상시킨다.

- `ThemeProvider`를 사용하여 애플리케이션 전역에서 테마를 설정하고, 컴포넌트 내에서 이를 쉽게 사용할 수 있다.

- 스타일의 범위가 컴포넌트로 제한되므로, 글로벌 네임스페이스 오염을 방지할 수 있다.

- props를 통해 동적으로 스타일을 변경할 수 있다. 이를 통해 조건부 스타일링이 간편해진다.

### styled-components의 장점

- 캡슐화된 스타일 : 컴포넌트별로 스타일을 정의하여 스타일 충돌을 방지할 수 있다. 이는 컴포넌트의 재사용성과 유지보수성을 높인다.

- 가독성 : CSS와 컴포넌트를 같은 파일에서 관리하므로 코드의 가독성이 향상된다. 스타일 정의와 컴포넌트 로직이 가까이 있어 이해하기 쉽다.

- 유연성 : props를 사용하여 동적으로 스타일을 변경할 수 있다. 조건부 스타일링이나 테마 적용이 간편해진다.

- 재사용성 : 스타일이 캡슐화되어 있어, 동일한 스타일을 여러 컴포넌트에서 재사용할 수 있다. 컴포넌트 간 스타일 공유가 쉬워진다.

- SSR 지원 : 서버 사이드 렌더링(SSR)을 지원하여, 초기 페이지 로딩 속도를 최적화할 수 있다.

- 테마 지원 : `ThemeProvider`를 통해 손쉽게 전역 테마를 적용하고, 유지보수할 수 있다. 다양한 테마를 적용하여 앱의 스타일을 쉽게 변경할 수 있다.

### styled-components의 단점

- 런타임 오버헤드 : 스타일을 런타임에 처리하므로, 성능에 영향을 줄 수 있다. 특히 대규모 애플리케이션에서 많은 스타일을 처리할 때 문제가 될 수 있다.

- 빌드 타임 오버헤드 : 빌드 프로세스에 추가적인 설정과 시간이 필요할 수 있다. 일부 빌드 도구와의 호환성 문제도 발생할 수 있다.

- 직관성 부족 : 전통적인 CSS 또는 SASS에 익숙한 개발자는 처음 사용 시 다소 낯설 수 있다. CSS 파일과 자바스크립트 파일을 분리해서 관리하는 방식과 다르다.

- CSS 기능 지원 제한 : 일부 고급 CSS 기능(예 : 미디어 쿼리, 애니메이션 등)은 기본적으로 지원되지 않거나, 제한적으로 지원될 수 있다. CSS 파일과 비교해 제한이 있을 수 있다.

- 스타일링 디버깅의 복잡성 : 생성된 클래스명이 난수화되어, 브라우저의 개발자 도구에서 스타일을 디버깅하는 것이 번거로울 수 있다.

보통 리액트 프로젝트에서는 단일한 방법을 사용한다.

</br>

## Tailwind CSS

Tailwind CSS는 유틸리티 퍼스트(Utility-First) CSS 프레임워크로, 미리 정의된 클래스들을 사용하여 빠르고 효율적으로 스타일링을 할 수 있게 해준다. 스타일을 작성할 때, 특정 요소에 직접 클래스를 적용하여 디자인을 구현한다. 이는 전통적인 방식과는 다르게 CSS 파일을 작성할 필요 없이, HTML 마크업에 스타일 클래스를 추가하는 방식이다.

### Tailwind CSS의 특징

- 유틸리티 퍼스트 : Tailwind CSS는 유틸리티 클래스를 중심으로 설계되었다. 각 클래스는 하나의 CSS 속성에 대응하며, 이를 조합하여 스타일을 적용한다.

- 모듈화 : 스타일을 재사용 가능한 작은 모듈로 분리하여 관리할 수 있다.

- 반응형 디자인 : Tailwind CSS는 반응형 디자인을 쉽게 구현할 수 있도록 돕는 다양한 유틸리티 클래스를 제공한다.

- 구성 가능성 : Tailwind의 설정 파일을 사용하여 프로젝트의 요구에 맞게 유틸리티 클래스를 커스터마이징할 수 있다.

- 빠른 프로토타이핑 : 미리 정의된 클래스들을 사용하여 빠르게 프로토타입을 만들 수 있다.

- 간결한 클래스 이름 : 짧고 의미 잇는 클래스 이름을 사용하여 코드의 가독성을 높인다.

### Tailwind CSS의 장점

- 생산성 향상 : 미리 정의된 유틸리티 클래스를 사용하여 스타일을 빠르게 적용할 수 있어, 생산성이 크게 향산된다.

- 일관된 디자인 : 프로젝트 전반에 걸쳐 일관된 디자인 시스템을 유지할 수 있다.

- 유연성 : 유틸리티 클래스를 조합하여 거의 모든 디자인을 구현할 수 있어, 매우 유연하다.

- 작은 CSS 파일 : 사용하지 않는 CSS는 제거되어 결과적으로 매우 작은 CSS 파일이 생성된다. 이는 페이지 로딩 속도를 향상시킨다.

- 미리 정의된 반응형 유틸리티 : 반응형 디자인을 쉽게 구현할 수 있는 다양한 유틸리티 클래스를 제공한다.

### Tailwind CSS의 단점

- 초기 학습 곡선 : Tailwind CSS의 유틸리티 클래스를 체계를 처음 접하는 개발자에게는 익숙해지기까지 시간이 걸릴 수 있다.

- HTML 마크업이 길어질 수 있음 : 유틸리티 클래스를 직접 HTML 요소에 추가하는 방식이기 때문에, HTML 마크업이 길어지고 복잡해질 수 있다.

- 특정한 디자인을 강제하지 않음 : Tailwind CSS는 아무런 디자인을 강제하지 않기 때문에, 처음 사용 시 일관된 디자인을 유지하기 어려울 수 있다.

- 커스터마이징 복잡성 : 설정 파일을 통해 클래스를 커스터마이징할 수 있지만, 이는 초보자에게는 다소 복잡할 수 있다.

### React와 함께 사용하는 경우의 장점

- 모듈화된 스타일링 : React 컴포넌트와 함께 Tailwind CSS를 사용하면, 각 컴포넌트에 필요한 스타일만을 적용하여 스타일링을 모듈화할 수 있다.

- 빠른 개발 속도 : 유틸리티 클래스를 사용하여 즉시 적용할 수 있으므로, 컴포넌트 개발 속도가 빨라진다.

- 일관된 스타일 적용 : Tailwind CSS를 사용하면 프로젝트 전반에 걸쳐 일관된 스타일을 적용할 수 있다. 이는 특히 대규모 팀에서 유용하다.

- 반응형 디자인 : Tailwind의 반응형 유틸리티 클래스를 사용하여 쉽게 반응형 디자인을 구현할 수 있다.

- CSS 파일 관리 필요 없음 : CSS 파일을 별도로 관리할 필요 없이, 컴포넌트 내에서 직접 스타일링을 처리할 수 있다.
